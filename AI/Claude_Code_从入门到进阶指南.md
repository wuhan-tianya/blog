# Claude Code：从入门到进阶的全方位指南

## 一、 什么是 Claude Code？
Claude Code 是一款由 Anthropic 官方发布、运行在命令行的终端原生 Agent 工具（CLI 工具）。不同于 Cursor 等基于 IDE 编辑器的 AI 工具，Claude Code 能够直接了解你的整个代码库、运行系统命令、执行测试并在发现问题时自动修复代码。

### 核心亮点：
1. **理解整个代码库**：无需手动粘贴代码，自动分析本地文件系统，支持大规模代码检索。
2. **终端原生交互**：直接执行 Bash 脚本、Git 提交、查看 `grep` 或 `ls` 等结果。
3. **闭环修复系统**：不仅仅写代码，它能通过执行单元测试来验证代码，报错后自主修正。
4. **模型支持**：原生搭载最强大的 `Claude 3.7 Sonnet`，支持顶级的代码推理能力与 Agent 行为。

## 二、 基础配置与安装

### 1. 前置环境要求
- **Node.js**：请确保已安装 Node.js（推荐 `>= 18.0.0` ）。
- **API 额度**：由于它直接消耗你的 Anthropic API 余额，你需要确保在 Anthropic Console 中绑定了支付方式或有足够的余额。

### 2. 安装命令
使用 npm 进行全局安装：
```bash
npm install -g @anthropic-ai/claude-code
```

### 3. 授权与启动
在终端进入你想进行代码开发的仓库目录，执行：
```bash
claude
```
首次执行会弹出一个网页用于 OAuth 登录（要求登录你的 Anthropic API 账号），授权完成后即可在终端与 Claude 开始交互。

## 三、 核心斜杠命令（Slash Commands）使用

在 `claude` 终端中，除了直接输入自然语言需求，掌握斜杠命令是最高效的做法。

| 命令 | 描述与使用场景 |
|---|---|
| `/help` | **查看帮助**。遇到不会用的功能时输入。 |
| `/clear` | **清空当前上下文**。AI 开始出现幻觉或之前聊得太多，输入此命令让它“失忆”，这非常重要，可以大大节省 Token 和成本。 |
| `/compact` | **压缩上下文**。当你不想完全 clear 会话，但 Token 已经积累很高时，这个命令会让 Claude 总结之前的关键思路，扔掉庞大的完整聊天记录。 |
| `/cost` | **查看成本**。实时打印出你在此会话（Session）中耗费了多少 Token，花了多少美元。随时掌握进度，避免破产。 |
| `/pr` | **创建 Pull Request**。当代码修改完成后，可以直接让它帮你生成一段精美的 PR 描述。 |
| `/bug` | 提交报错反馈给官方。 |

## 四、 核心工作流：入门到进阶实战

### 阶段 1：全局代码检索与问答（入门）
*使用场景*：接手一个全新的开源项目或旧代码库，需要快速上手。
*你可以这样输入*：
> "帮我分析一下当前项目的架构，前端和后端分别在哪个目录？"
> "找出所有涉及用户鉴权（Authentication）的逻辑，并解释其实现流程。"

*Claude 的动作*：它会自行使用 `grep`、`find` 或 `ls` 等系统命令搜寻目录，然后给出精准答案。

### 阶段 2：定向修改与功能开发（进阶）
*使用场景*：新增小功能，或重构特定模块。
*你可以这样输入*：
> "在 `src/components/Header.jsx` 中新增一个暗黑模式的切换按钮。请确保样式与 Tailwind 现有配置匹配。"

*Claude 的动作*：
1. 它会使用 `cat` 或者自定义的工具读取 `Header.jsx`。
2. 它会写入代码并使用工具（如 `sed` 或专用覆写工具）保存文件。
3. 如果涉及到未安装的包，它会问你：“是否允许我运行 `npm install xxx`？”（这里需要你在终端中输入 `y` 批准，非常安全）。

### 阶段 3：Agentic Loop 闭环排错（高级）
*使用场景*：你有一个破损的测试，或是一个复杂的依赖报错，需要它自行解决。
*你可以这样输入*：
> "运行 `npm run test`，把所有报错的用例都修好。"

*Claude 的动作*：
它将进入一种**自我博弈和修复**的循环模式。
1. 执行测试。
2. 看到红色的 Error 报错。
3. 思考：为什么报错？去读取相关文件。
4. 修改代码。
5. **再次执行测试验证**。
6. 重复 1-5 步，直到控制台绿灯（Pass），并最后回复你："测试已全部通过。"

## 五、 高级技巧与排坑指南

### 1. 管理好你的成本（Cost Management）
因为基于 API 计费，如果让它在极其庞大的代码库里胡乱使用 `grep` 搜索，消耗会非常大。
- **最佳实践**：每完成一个小需求，及时使用 `/compact` 或 `/clear` 缩减上下文。不要在一个会话里持续对话几个小时。

### 2. 使用 `.claudesignore` 忽略垃圾文件
就像 `.gitignore` 一样，你可以在项目根目录建一个 `.claudesignore` 文件。把不想让它读取的、庞大的生成文件放进去，例如：
```text
dist/
build/
node_modules/
package-lock.json
*.svg
```
这样它在遍历文件夹时就会自动避开，提升速度、省 Token。

### 3. 主动提供指令上下文（Context Files）
如果你有一个架构设计文档或 API 规范文档（如 `DESIGN.md`），可以在首次提问时顺便指引它：
> "我要开发登录模块，请先阅读 `DESIGN.md` 中的规范，然后开始编写代码。"

### 4. 工具权限设置 (Permission)
由于 Claude 具备运行 shell 脚本的能力，为了安全，它在删除文件或安装依赖前会高亮请求你的批准（Require Approval）。请务必**仔细审查**它将要执行的 Shell 命令，确保不会删除重要数据。
